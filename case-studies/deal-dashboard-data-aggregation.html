Deal Dashboard - Data Aggregation

Deal Dashboard

Problem

The deal data from various SSP’s and corresponding validation data from the client database were in different platforms and there was no single way of collecting data in a single platform and comparing them. The corresponding goal in salesforce needed to be attributed to the deal so that the deal could be optimized for pacing evenly and revenue tracked.

Challenges

  • Data was in different formats across each of the platforms with some platforms using different terminology
  • Data was available at different times of the day.
  • Data had no visual representation.
  • Salesforce data had different tracking mechanisms and had to attribute corresponding deals.

Tools

  • AdJuster

  • TapClicks

  • Domo

  • Google BigQuery

  • Google Data Studio

  • Salesforce

Result

  • AdJuster/TapClicks dashboard to create visual representation.
  • Data collected using API’s and stored in Google Cloud and moved to dashboard.
  • AdJuster/Tapclicks setup to update data at specific times to work around the time zone issue.
  • Transit excel doc created to lookup Salesforce ID against deal ID
  • Salesforce data attributed to Deals using transit doc.
  • Data shown in a singular view.
  • Data representation available NBD at 12 noon.
  • Enable’s optimization of deal.
  • Client was able to notice discrepancy earlier.
